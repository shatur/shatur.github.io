Overview Of The Project:

         Now coming to our project, our aim was to implement some of the APIs of Web-Extension on android. But due to some difficulties I faced during implementing this project, I am able to implement only one of the API, ie, browisngData. It enables extensions to clear private data. There are various methods of browsingData API like removeHistory, removeCookies, removeCache etc which helps us to remove specific data from the user profile. I will be writing a blog, on how one can use these methods in their addons.

Difficulites Faced:

         There were some difficulties which we faced during the implementation of this project. I started with the implementation of settings method, which enable addons to get user preferences for clearing browsing data.We faced some issues but not so difficult to solve.

    Then we moved on to implementing the removeHistory method. This was the one where we faced a quite hurdles. On firefox for android, we store history in SqliteDB, we've two tables, one is history and another one is visits. For every history items, we can have many visits. We've local and remote visits, which we used to weight sites for top sites, highlights and all, to display on our home page. In addition, we need to perform all these operations in sync safe manner. While removing history items, we do not want to mess our sync operation and check it does not suffer from performance or concurrency issues. The bug is still in review and work is being done on it. Others were not so difficult. Some of them have been landed and some are in process to land.
